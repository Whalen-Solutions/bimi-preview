LLM Prompt for Email Content

Copy this prompt and send it to any small language model (Haiku, Gemma, Phi, Mistral, etc.) along with the company name, domain, and industry. The model will return structured JSON that you can pass directly to the preview template.

Prompt

Generate realistic email preview content for a BIMI inbox mockup showing how a company's verified logo appears in Gmail.

Inputs:
- Company name: {company_name}
- Email domain: {domain}
- Industry: {industry}

Return ONLY a JSON object with these fields:

{
  "inbox_subject": "Subject line for the inbox list (50-80 chars)",
  "inbox_preview": "Preview snippet shown after the subject (60-100 chars)",
  "email_body": "2-3 short paragraphs of HTML using only <p> tags. Professional, warm tone. Reference the company by name. Include a specific, plausible reason for the email that fits what this company actually does.",
  "other_senders": [
    {"name": "A plausible colleague or contact", "subject": "Realistic subject", "preview": "Preview text", "time": "e.g. 9:15 AM"},
    {"name": "Another contact", "subject": "Subject", "preview": "Preview", "time": "e.g. Yesterday"},
    {"name": "A lesser-known service or noreply sender without its own Gmail logo (e.g. Teamwork, Gusto, DocuSign)", "subject": "Notification-style subject", "preview": "Preview", "time": "e.g. Mar 28"}
  ]
}

Rules:
- Reflect what {company_name} actually does — not generic industry stereotypes
- other_senders should look like normal inbox neighbors (personal + notifications)
- email_body HTML: only <p> tags, no other elements
- No email addresses as literal text in the output
- Today is {today}. Time references must be current — no past quarters, months, or years unless today is near their end. Use the company's fiscal calendar for financial content if known, calendar dates for events
- Return ONLY valid JSON, no markdown fences, no explanation

How to Use

1. Replace {company_name}, {domain}, {industry}, and {today} with your values.

2. Send the prompt to your preferred LLM.

3. Parse the returned JSON. The server flattens other_senders into numbered template keys (e.g. other_sender_1_name) and derives each sender's initial from their name. Email times (inbox_time, email_time) and the phone status bar time are computed server-side in the user's timezone.

For example, in the Flask app you would add the parsed JSON fields to the render_template() call: 

import json

# llm_response = call_your_llm(prompt)
content = json.loads(llm_response)

return render_template(
    "preview.jinja2.html",
    company=company,
    domain=domain,
    industry=industry,
    bimi_svg=bimi_svg,
    job_id=job_id,
    sender_email_parts=sender_email_parts,
    **content,   # spreads all the LLM-generated fields into the template
)
